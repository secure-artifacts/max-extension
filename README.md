# MAX 出勤记录器（Chrome 浏览器插件）

在 **MAX 网页版** 通话中，自动记录**每个参会者的上线 / 下线时间和在线时长（分钟）**。
适合用来统计会议/通话的人员出勤情况。所有数据只保存在你本地浏览器，不上传任何服务器。

## 工作原理（全自动）

插件**自动**识别参会者，**不需要手动点选**。它同时用三种来源，自动选最可靠的一种：

1. **自动·页面**：自动定位通话界面里的参会者条目（在画面/名单里=在线，消失=下线）。
2. **自动·网络**：挂钩通话自身的 WebSocket / 接口数据，直接读取参会者姓名与进出事件。
3. **手动校准**（可选兜底）：万一前两种都不准，点一下任意一位参会者即可，右键「手动校准」按钮可清除恢复全自动。

是否在通话中由 **WebRTC 连接状态**自动判断（事件驱动）：**接通自动开始、挂断自动结束**。

### 可靠性（边做别的事也不丢记录）

- 通话的开始/结束是**事件驱动**的，不依赖你停留在通话页面。
- 通话进行中页面在播放音频，浏览器不会冻结后台标签页的计时，因此你切到别的标签、最小化窗口，记录都会继续。
- 每 5 秒自动落盘一次，意外关闭也能保留到最近状态。
- **一通电话 = 一条记录**：挂断即结束当前记录，下一通自动开启新记录。

> 提示：MAX 改版导致某次识别不准时，用「手动校准」点一位参会者即可临时纠正。

### 调试模式（识别不到人时）

如果某次完全识别不到参会者，可在通话页按 F12 打开控制台，执行：

```js
localStorage.setItem('maxct_debug','1')
```

然后刷新页面。控制台会打印出 `[MAXCT:ws]` / `[MAXCT:fetch]` 等抓到的数据样本。把其中包含人名的一条发我，我可以据此把「自动·网络」识别做到精确。关闭调试：`localStorage.removeItem('maxct_debug')`。

## 安装方法（加载已解压的扩展程序）

1. 打开 Chrome，地址栏输入 `chrome://extensions` 回车。
2. 打开右上角 **「开发者模式 / Developer mode」**。
3. 点击 **「加载已解压的扩展程序 / Load unpacked」**。
4. 选择本文件夹 `d:\redownload\ruanjian`（含 `manifest.json` 的目录）。
5. 工具栏出现蓝色时钟图标即安装成功。

## 使用步骤

1. 打开并登录 MAX 网页版，加入通话，例如：
   `https://max.ru/joincall/m0G5catfx2AQ7znq80o2PsUp1AznpnDd7GPxZalgNho`
   （会跳转到 `web.max.ru` 进行通话）
2. 通话接通后，右上角会出现悬浮窗「MAX 出勤记录」，**自动开始计时并识别参会者**。
3. 悬浮窗实时列出每个人和各自的在线时长（在线者绿色置顶；有人离开变灰，回来重新累计）。右上角小标签显示当前识别来源（自动·页面 / 自动·网络 / 校准）。
4. 你可以同时去做别的事、切到别的标签页，记录会一直进行；**挂断后自动保存**这一通的记录。
5. 点工具栏图标可查看每场通话、每个人的在线分钟数，并 **导出 CSV**。
6. （可选）若某次自动识别不准，点悬浮窗「手动校准」再点一位参会者纠正；右键该按钮可清除校准、恢复全自动。

## 悬浮窗按钮说明

- **手动校准**：可选兜底。点一下进入点选模式，再点一位参会者来纠正识别（Esc 取消）；**右键**此按钮可清除校准、恢复全自动。
- **开始 / 结束**：手动开始或结束一次记录（自动检测失效时使用）。
- **—**：折叠；**×**：隐藏（刷新页面后会重新出现）。可拖动标题栏移动位置。

## 工具栏弹窗

- 顶部：当前通话时长、当前在线人数。
- 列表：每一场通话可展开，查看每位参会者的在线时长（在线者绿色置顶）。
- **导出CSV**：导出「通话开始/结束、参会者、在线分钟数、首次/最后出现时间」。
- **校准**：等同于悬浮窗的校准（会切回当前通话标签页操作）。
- **清空**：删除全部记录。

## 文件结构

```
manifest.json            扩展配置（Manifest V3）
inject.js                注入页面，挂钩 WebRTC 判断通话是否进行中
content.js               核心：参会者识别/校准 + 出勤计时 + 悬浮窗 + 存储
overlay.css              悬浮窗样式
background.js            后台 Service Worker：工具栏角标
popup.html/.css/.js      工具栏弹窗：每场通话的人员在线时长 + 导出
icons/                   扩展图标
tools/generate_icons.js  重新生成图标（可选）
```

## 隐私说明

- 插件仅在 `max.ru` / `web.max.ru` 下运行。
- 不读取通话内容，只读取参会者**显示名称**与「是否在名单中」，用于统计在线时长。
- 所有记录保存在 `chrome.storage.local`（你本机），不上传。

## 常见问题

- **识别不到人 / 名字不对？** 先按上面的「调试模式」抓一条数据发我，我把「自动·网络」做精确；临时也可用「手动校准」点一位参会者纠正。
- **没有自动开始？** 用悬浮窗「开始」手动记录；或刷新通话页。
- **有人短暂掉线又回来？** 默认连续约 4 秒未出现才算「下线」，回来后会累加在线时长（分多段统计，CSV 里体现为总时长）。
- **想调参数？** 在 `content.js` 顶部修改：`SCAN_MS`（扫描间隔）、`MISS_SCANS`（判定离线的连续缺席次数）、`GRACE_MS`（通话断线宽限）、`MIN_SESSION_MS`（最短记录）、`NET_TTL_MS`（网络数据有效期）。
